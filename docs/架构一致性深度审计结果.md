  4.3 架构禁忌验证

  Architecture §2.5 "Repository 原则" + §7 ViewModel 规则隐含约束：
  - ViewModel 用 viewModelScope；Worker 用 coroutineScope；Repository 不启动协程、不切线程
  - 一旦 Repo 内出现 withContext(ioDispatcher) { ... } → 同时违反"不启动协程"和"不做线程决策"

  这是经典 "可疑的注入即未来的反模式"：依赖图上存在但当前未用的字段，会在下一个 Task（3.6 实现 Repository 真实方法体时）被 AI 顺手 wrap 进 withContext，从而架构破缺正在被预先安装。

  4.4 推荐：Option A（强制移除）

  理由

  1. 零使用价值：当前与可预见未来（Phase 3-8）均无线程切换需求
  2. 强诱导效应：留下 @Suppress 注入即为后续违反 Arch §2.5 提供了完整模板
