架构核心原则
整体采用单模块 + 包结构分层，不引入 Gradle 多模块。数据流方向严格单向：Room → Repository → ViewModel → Compose UI，禁止反向依赖。
几个关键设计选择

Widget 同步：没有直接监听 Flow 刷新 Widget（太耗电），而是在 Repository 写入后手动触发 WidgetSyncWorker，Widget 用独立的 WidgetDataProvider 做一次性查询，与主流程解耦。
UiState + Channel 双轨：持久状态（列表、加载中）放 StateFlow<UiState>，一次性事件（导航、Toast）走 Channel<UiEvent>，避免重组时重复触发副作用。
Repository 职责边界：只做数据聚合和 Entity↔Model 映射，不含任何 if/when 业务判断逻辑，过滤/排序逻辑在 ViewModel 的 combine 中处理。
禁止 UseCase 层：个人项目 Repository 已足够薄，强行加 UseCase 只会增加文件数量而不增加可维护性。
每模块 README 模板：包含「AI 开发提示」字段，帮助 AI 协同时快速定位修改范围。
